# 二维码分片扫描合成工具

Languages: [简体中文](./README.md) · [English](./docs/README_en.md)

一个利用 **Word VBS + Android 原生 APP** 实现文件跨设备传输的工具。核心思路是：把任意文件转换成 Base64，再通过 Word 的「文字转二维码」功能切分成多张二维码图片，最后用手机连续扫描并还原成原文件。

## 项目初衷

这个项目诞生于一次特殊的场景需求：

> 在培训机构的电脑机房里，想要导出半导体电路图的 PDK 技术库。但机房的电脑做了严格限制——手机通过 USB 线接入后只能充电，无法进行 USB 文件传输；U 盘同样被禁用。尝试进入 BIOS 修改启动项、用微 PE 系统拷贝数据，结果发现 BIOS 被加锁，无法修改启动顺序。

于是这个软件应运而生：利用机房电脑上已有的 **Office 2016**，通过 Word 的 **DISPLAYBARCODE 字段** 把文件编码成二维码，再用手机上的原生 APP 连续扫描，最终把技术库文件从受限制的机房环境"搬"到虚拟机外的 Windows 系统中。

## ✨ 功能特性

### VBS 二维码生成器

- ✅ **Word 内一键生成**：在 Word 文档中直接生成二维码图片，无需额外软件
- ✅ **Base64 分片编码**：自动将文件切分为适合二维码容量的数据块
- ✅ **统一校验码**：每页二维码头部包含相同校验码，防止扫描时混入其他二维码
- ✅ **原文件名还原**：文件名通过 Base64 编码传输，APP 端自动还原原始文件名和后缀
- ✅ **UTF-8 中文支持**：中文文件名经过 Base64 编码，避免 Word 字段不支持非 ASCII 字符的问题
- ✅ **自动分页**：根据数据量自动计算页数，第一页携带文件名，后续页只携带数据

### Android 原生 APP

- ✅ **原生开发**：使用 Kotlin + CameraX 原生实现，非 WebView 套壳
- ✅ **连续扫描**：自动识别分片序号，支持乱序扫描和补扫
- ✅ **校验码过滤**：只接收与当前任务校验码匹配的二维码，避免误扫
- ✅ **草稿箱续扫**：按返回键可保存当前进度到草稿箱，下次继续扫描
- ✅ **原文件名保存**：自动还原 VBS 端传入的原始文件名和后缀
- ✅ **保存到 Download/QR**：文件保存到手机下载目录的 `QR` 子文件夹中
- ✅ **微信风格路径提示**：保存成功后提示 `文件已保存到 /sdcard/Download/QR/文件名`
- ✅ **文件大小显示**：结果页显示已还原文件大小，小于 1KB 显示字节数
- ✅ **多摄像头切换**：支持切换后置不同摄像头
- ✅ **大二维码支持**：优化内存和解析逻辑，支持较大容量的二维码

## 🚀 使用方法

### 1. 在电脑上生成二维码

1. 打开 Word 2016（或支持 `DISPLAYBARCODE` 字段的版本）
2. 按 `Alt + F11` 打开 VBA 编辑器
3. 插入模块，粘贴 `QR生成-v6.vbs` 中的代码
4. 运行 `QR` 宏，选择要传输的文件
5. Word 会自动生成多页二维码文档
6. 全选复制或导出为图片，在受限制电脑上展示这些二维码

### 2. 在手机上扫描还原

1. 在 Android 手机上安装 `QR合成-原生-v7.apk`
2. 打开 APP，对准电脑屏幕上的二维码连续扫描
3. APP 会自动识别校验码、序号和总数
4. 扫描完成后点击「保存文件」
5. 文件会自动保存到 `/sdcard/Download/QR/` 目录下

## 🛠 技术细节

### 二维码协议格式

```
TZ|checksum|index|total|fileNameB64|data
```

| 字段 | 说明 |
|---|---|
| `TZ` | 固定协议头 |
| `checksum` | 校验码，由文件名 Base64 前 6 位 + Base64 数据总长度组成 |
| `index` | 当前页序号（从 1 开始） |
| `total` | 总页数 |
| `fileNameB64` | 文件名 Base64 编码，仅第 1 页携带 |
| `data` | 当前页的 Base64 数据块 |

### 分片策略

- 后续页数据固定 1000 字符
- 第 1 页数据长度动态计算，并对齐到 4 的倍数，保证整体 Base64 可解码
- 文件名通过 Base64 编码传输，支持中文和特殊字符

### Android 保存机制

- Android 10+ 使用 MediaStore 写入 `Download/QR` 目录
- 返回真实文件路径 `/storage/emulated/0/Download/QR/文件名`
- 提示时替换为 `/sdcard/Download/QR/文件名`
- 通过文件魔数识别扩展名（JPEG/PNG/GIF/PDF/ZIP/BMP 等）
